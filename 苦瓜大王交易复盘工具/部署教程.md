# 部署到 GitHub Pages 教程

> 🎯 目标：把你电脑上的「苦瓜大王交易复盘工具」发布成一个可以通过网址访问的网页
> 💰 **免费** · 无需服务器 · 全球可访问

---

## 第一步：注册 / 登录 GitHub

**GitHub** 是一个代码托管平台，GitHub Pages 是它提供的免费网页托管服务。

1. 浏览器打开 https://github.com
2. 如果没账号，点 **"Sign up"（注册）** 创建账号
   - 输入用户名、邮箱、密码
   - 验证邮箱即完成注册
3. 已有账号则直接 **"Sign in"（登录）**

> 💡 GitHub 可以根据你的浏览器语言显示中文界面。如果你用 Chrome 且系统语言为中文，登录后界面会显示为中文。

---

## 第二步：创建新仓库

1. 登录后，点击右上角 **+** 号 → **"New repository"（新建仓库）**

2. 填写以下信息：
   - **Repository name（仓库名称）**：`trade-lens`（或任意名字，如 `kugua`）
   - **Description（描述）**：`苦瓜大王交易复盘工具`（可选）
   - **Public / Private**：选 **Public（公开）**（Private 也可以，但 Pages 功能需付费）
   - **不要勾选** "Initialize this repository with a README"
   
3. 点击 **"Create repository"（创建仓库）**

---

## 第三步：上传文件

### 方法一：网页上传（最简单，推荐）

1. 进入刚创建的仓库页面
2. 点击 **"uploading an existing file"（上传已存在文件）** 链接
3. 在电脑上打开文件夹 `C:\Users\Xiaomi\Documents\交易复盘工具`
4. **全选里面所有文件和文件夹**
5. 拖拽到 GitHub 的上传区域
6. 在页面底部填写提交信息（Commit message）：`初始部署`
7. 点击 **"Commit changes"（提交更改）**

### 方法二：Git Bash 命令上传（适合熟悉命令行的用户）

```bash
# 先安装 Git: https://git-scm.com/downloads/win

cd "C:\Users\Xiaomi\Documents\交易复盘工具"

git init
git add .
git commit -m "初始部署"

git branch -M main
git remote add origin https://github.com/你的用户名/trade-lens.git
git push -u origin main
```

---

## 第四步：开启 GitHub Pages

1. 进入你的仓库页面，点击顶部 **"Settings"（设置）**

2. 在左侧菜单找到 **"Pages"（页面）** 并点击

3. 在 **"Branch"（分支）** 下拉菜单中：
   - 选择 `main`
   - 文件夹选 `/ (root)`
   
4. 点击 **"Save"（保存）**

5. 等待约 1-2 分钟，页面顶部会出现绿色提示框：
   ```
   Your site is live at https://你的用户名.github.io/trade-lens/
   ```

---

## 第五步：访问你的网站

点击上面出现的网址，或者在浏览器输入：

```
https://你的用户名.github.io/trade-lens/
```

如果看到白屏或 404，**稍等几分钟再刷新**，GitHub Pages 有时需要一点时间生效。

---

## 后续更新

修改了电脑上的文件后，想同步到网页：

1. 进入仓库页面
2. 点击 **"Add file"** → **"Upload files"**
3. 重新拖入所有文件（覆盖旧的）
4. 提交即可，GitHub Pages 会自动更新（约 1-2 分钟）

## 常见问题

### Q: 打开网址后显示 404
A: 等 2 分钟再刷新。如果还是 404，检查仓库 Settings → Pages 里的 Branch 是否设置为 `main`，且文件夹为 `/ (root)`。

### Q: 能否用中文域名？
A: GitHub Pages 不支持自定义中文域名，但网站内容本身是中文，域名用英文即可。

### Q: 数据安全吗？
A: 你的交易数据存储在**浏览器本地**（localStorage），不会上传到 GitHub，非常安全。GitHub Pages 只托管这个 HTML 文件本身。

### Q: 为什么 AI 分析用不了？
A: 需要先在工具设置中配置 DeepSeek 或 OpenAI 的 API Key。AI 分析是直接在浏览器里调用 API 的，不经过任何服务器。
